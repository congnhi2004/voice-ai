## Configuration

Local/default behavior:

```env
STORAGE_PROVIDER=local
AUDIO_STORAGE_MODE=local
VIDEO_STORAGE_MODE=local
AUDIO_STORAGE_DIR=/app/data/audio
AUDIO_BASE_URL=http://localhost:8080/audio
```

Production GCS behavior:

```env
STORAGE_PROVIDER=gcs
GCS_AUDIO_BUCKET=voice-ai-prod-audio
GCS_ARTIFACT_BUCKET=voice-ai-prod-artifacts
GCS_AUDIO_PREFIX=voice-ai/audio
GCS_SOURCE_VIDEO_PREFIX=voice-ai/video/source
GCS_INTERMEDIATE_PREFIX=voice-ai/video/intermediate
GCS_RENDERED_VIDEO_PREFIX=voice-ai/video/rendered
SIGNED_URL_TTL_SECONDS=3600
```

`AUDIO_STORAGE_MODE=gcs` and `VIDEO_STORAGE_MODE=gcs` also work as per-surface overrides if `STORAGE_PROVIDER` remains `local`.

## Behavior

- Local TTS still writes to `AUDIO_STORAGE_DIR` and returns `/audio/...` URLs.
- GCS TTS writes a local scratch copy for existing MLflow/audio-path compatibility, uploads generated bytes to `gs://GCS_AUDIO_BUCKET/GCS_AUDIO_PREFIX/...`, and returns a signed GET URL.
- Local video artifacts still return `/v1/video-localization/jobs/{job_id}/artifacts/{filename}` API URLs.
- GCS video artifacts upload source, intermediate, subtitle, voiceover, and rendered files under configured prefixes and return signed GET URLs.
- Status reads regenerate signed URLs so expired URLs in persisted status do not become permanent dead links.
- Legacy artifact download routes redirect to a fresh signed URL when the status artifact path is `gs://...`.
